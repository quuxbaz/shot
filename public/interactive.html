How to use it?

The API has two API endpoints:

  1. /shot/image
  2. /shot/pdf
Let's capture a fast-loading website. Using the /shot/image endpoint we produce an image of the website. Using the /shot/pdf endpoint we produce a PDF with the image of the website embedded in it.

How should I use it on my website?

Take a look at the source of this page and notice the URL in the a-href tags are url-encoded. That's because the symbols "://" cannot travel on a browser URL. To avoid having to translate that always by hand, you can use the Javascript function encodeURIComponent as in this case and this case. Here's the relevant code:

<a href="" id="link1" target="_blank">View PDF version</a> <script src="https://code.jquery.com/jquery-3.3.1.min.js" integrity="sha256-FgpCb/KJQlLNfOu91ta32o/NMZxltwRo8QtmkMRdAu8=" crossorigin="anonymous"></script> <script> var url = "https://varvy.com/pagespeed/wicked-fast.html"; $(document).ready(function () { $("#link1").attr("href", "/shot/pdf/" + encodeURIComponent(url)); }); </script>
